Expand description

Server to handle service requests from masters from the master communication channel.

The SlaveReqHandler acts as a server on the slave side, to handle service requests from masters on the master communication channel. It’s actually a proxy invoking the registered handler implementing VhostUserSlaveReqHandler to do the real work.

The lifetime of the SlaveReqHandler object should be the same as the underline Unix Domain Socket, so it gets simpler to recover from disconnect.
